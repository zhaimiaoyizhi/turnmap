Node Resize Module PRD

Owner scope: user-controlled node dimensions, persistence, and interaction safety.

Requirements

  • Expose persistent resize drag zones on the left edge, right edge, bottom edge, lower-left corner, and lower-right corner.
  • Do not expose top-edge or upper-corner resize zones in 0.7.0.
  • Precedence: Rebuild > user manual resize > refresh/deep scan recovery > automatic layout.
  • Manual dimensions must survive refresh, deep scan, conversation switch, browser restart, and automatic layout changes.
  • Show Original / Show Expanded may auto-switch normal/expanded size only before the user manually resizes that node.
  • Enforce minimum and maximum dimensions so saved layouts remain usable in Side Panel and Full Page views.
  • Resize should update React graph state and rely on the existing debounced graph save pipeline rather than writing immediately on every drag movement.
  • Dragging completion may show a localized status-bar message such as node size saved.
  • Resize drag zones are persistent but visually low-noise by default. Their affordance should become clearer on hover, keyboard focus, or node selection.
  • User node dimensions are shared across layout modes. Do not maintain separate manual sizes for Single-side, Radial, Matrix, and Two-sided layouts in 0.7.0.

Out Of Scope

  • Freeform resizing from the top edge or top corners.
  • Global layout algorithm rewrite.
  • Per-layout independent node sizes unless explicitly needed later.

Documentation

User guide must explain that nodes can be resized from the left edge, right edge, bottom edge, lower-left corner, and lower-right corner.

Acceptance

  • Tests cover dimension save/restore, refresh/deep scan remap migration, min/max clamping, and Rebuild reset behavior.
  • Resize interactions do not break node dragging, selection, multi-select, text edit, right-click source jump, or expansion controls.
  • PNG/SVG/Canvas export reads current visible dimensions where relevant.
