Node Expansion Module PRD

Owner scope: API-only expansion of one AI answer into a compact, editable mini-map inside the original node.

Core Decisions

  • API-only in 0.7.0. No local rule-based fallback.
  • AI expansion uses original assistant answer/source text when available; summary fallback is allowed and logged.
  • The user question may be sent as context to improve decomposition quality, but the output mini-map must decompose only the assistant answer and must not turn the user question into mini nodes.
  • Mini-node titles should follow the original answer language. Node expansion is content structuring, not translation. UI labels and messages still follow TurnMap interface language settings.
  • Expansion data is stored separately from the outer node title/summary.
  • AI output writes only after request success and structure validation. Failures leave the node unchanged.
  • Mini nodes have title only. No mini summary fields in 0.7.0.
  • Preferred mini-node count: 6-18. Hard maximum: 24.
  • Documentation must disclose that node expansion sends the selected answer, and possibly the user question as context, to the configured AI provider. Do not show a confirmation before every expansion; use disabled/configuration guidance when no provider is configured.
  • Documentation must present API-only expansion as an intentional 0.7.0 limitation, not as a broken disabled feature.
  • Cancellation is out of scope for single-node expansion in 0.7.0. While expansion is running, disable duplicate expand/re-expand actions, show progress, and rely on atomic no-write behavior for failures.

Mini-Map Editing

  • Required: edit mini-node title, color mini nodes, mark mini nodes important, delete mini nodes, remove related internal links, save/restore edits.
  • Deferred: mini summaries, subtree fold/unfold, manual mini-node creation, manual internal linking, internal link edits, freeform mini-node dragging.
  • If fewer than two mini nodes remain after deletion, keep expanded state and show localized guidance.

Export

  • TurnMap JSON must preserve full expansion data and current display mode.
  • PNG must render currently visible expanded mini-map state.
  • Obsidian Canvas must export outer node plus editable nearby mini nodes and internal links.
  • Canvas import does not need to round-trip expansion data in 0.7.0.

Acceptance

  • Mocked AI tests cover success, no API key, provider failure, invalid structure, hard cap, atomic no-write failure, and summary fallback logging.
  • Save/restore tests cover display mode, mini-node edits, deleted mini nodes, and outer title/summary preservation.
  • All UI/status/task-log copy is localized and available to AI custom translation packs.
