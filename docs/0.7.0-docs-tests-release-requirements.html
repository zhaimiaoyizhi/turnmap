Docs, Tests, And Release Module PRD

Owner scope: documentation, localization completeness, verification, and optional packaging for 0.7.0.

Documentation

  • Update README.md, README.zh-CN.md, CHANGELOG.md, and docs/user-guide.md.
  • Emphasize that TurnMap JSON is the preferred format for complete save/restore fidelity.
  • Document resize drag zones: left edge, right edge, bottom edge, lower-left corner, lower-right corner.
  • Document API-only node expansion, no local fallback, Show Original vs Delete Expansion, and Re-expand overwrite confirmation.
  • List API-only node expansion as a known 0.7.0 limitation so users understand that a configured AI provider/API key is required.
  • Document Obsidian Canvas as editable external export, not full round-trip persistence.

Testing

  • Unit coverage for node expansion save/restore, mini-node edits, dimensions, topic group save/restore, batch tags, batch link edits, export behavior, and i18n completeness.
  • Mock AI provider behavior; tests must not depend on live API keys.
  • After each implemented subfeature, run npm.cmd run test:unit and npm.cmd run typecheck.
  • At the end, run npm.cmd run build and npm.cmd run package.

Localization

  • Every new dialog, confirmation, inline notice, option label, placeholder, status-bar message, task-log entry, disabled hint, and error must have English and Chinese keys.
  • Custom AI language packs must inherit the new keys through the existing 0.5.x language-pack mechanism.
  • Avoid browser-native prompt text for new flows.

Release Boundary

  • Do not commit or push unless explicitly requested.
  • Do not publish a release unless explicitly requested.
  • After Codex verification passes, produce a local testable unpacked package under release/, plus the corresponding zip if the packaging script creates it.
