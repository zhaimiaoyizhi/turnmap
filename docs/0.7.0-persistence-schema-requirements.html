Persistence And Schema Module PRD

Owner scope: shared storage/import/export schema foundations for 0.7.0 features.

Responsibilities

  • Extend saved graph schema for node dimensions, manual-size flags, expansion data/display mode, topic group records, and any generated proxy-edge metadata.
  • Preserve existing positions, user edges, node overrides, custom nodes, hidden nodes, source anchors, hidden root, hidden auto edges, and layout mode.
  • Support confident node-id remap migration for dimensions and expansion state when refresh/deep scan identifies the same original node.
  • Ensure TurnMap JSON exports/imports full 0.7.0 state with complete save/restore fidelity.

Open Questions

Schema Version

Decision: bump saved graph schemaVersion from 2 to 3 for 0.7.0. Version 3 owns node dimensions, manual-size flags, expansion data/display mode, topic group records, and generated proxy-edge metadata. Version 2 data must continue to load through compatibility defaults.

Import Compatibility

Decision: prefer forward-looking 0.7.0 behavior over preserving every preview-era quirk. Older graphs without dimensions, expansion data, or topicGroups should load with compatibility defaults. Legacy custom-topic notes that can be confidently identified as old Collapse Topic output should be migrated into schemaVersion 3 topic group records when enough hidden-node/source metadata exists. If confidence is low, keep the legacy note as a normal custom note rather than risking a wrong group. High-confidence migration requires: id matching the old custom-topic-* pattern, tags containing topic, summary matching the old "Collapsed topic from N turns." structure, and hiddenNodeIds containing the original hidden nodes. sourceAnchors matching member nodes are useful supporting evidence but not a hard requirement.

Topic Group Snapshot Scope

Decision: topic groups store a minimal restoration snapshot, not a full copy of node content. Store member ids, member positions before collapse, member hidden state before collapse, related user-edge ids/payload needed for restoration, and editable topic metadata. Do not duplicate node title, summary, source text, or sourceAnchors inside the group record; those remain owned by nodeOverrides/customNodes/turn data to avoid two sources of truth.

Acceptance

  • Unit tests cover save/load/JSON import/export for dimensions, expansion data, mini edits, topic groups, proxy edge exclusion, and sourceAnchors preservation.
  • Older saved graphs still load without data loss.
  • Full fidelity recovery is documented as TurnMap JSON only.

Risk: adding several 0.7.0 fields independently can create partial persistence bugs. This module should land before feature UI work.
