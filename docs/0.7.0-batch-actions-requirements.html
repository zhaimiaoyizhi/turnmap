Batch Actions Module PRD

Owner scope: localized multi-node tag management and true multi-link batch editing.

Node Tags

  • Integrate add/remove tag controls into the existing Node Actions panel.
  • No new browser-native prompt dialogs for tag workflows.
  • Show the selected nodes' tag union; removing a tag removes it from every selected node that has it.
  • Normalize tags lightly: trim, max 32 characters, de-duplicate per node, preserve case and non-English text.
  • Tags persist and continue feeding topic analysis/link suggestions.
  • Removing a tag from selected nodes does not require confirmation. Use localized status-bar feedback such as how many nodes were updated.
  • A batch tag add/remove operation should behave as one undo/history step, not one step per changed node.

Link Batch Mode

  • Batch link operations are triggered by selecting multiple links, not by selecting multiple nodes.
  • Link Actions supports batch relationship type and importance edits.
  • Relationship type drives color. Do not add independent custom link color overrides in 0.7.0 unless already supported by the existing model.
  • Multi-selecting nodes must not silently modify links.
  • A batch link edit should behave as one undo/history step, not one step per changed link.

Localization

Every label, placeholder, disabled hint, status message, and error in this module must be added to English and Chinese i18n maps and covered by i18n completeness tests so AI custom translation packs inherit the keys.

Acceptance

  • Unit tests cover tag add/remove persistence, normalization, tag union behavior, i18n key completeness, link batch type/importance changes, and type-driven colors.
  • Fast local edits use status-bar feedback, not API task-log spam.
