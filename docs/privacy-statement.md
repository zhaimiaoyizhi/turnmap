# TurnMap Privacy Statement

TurnMap is an Edge extension that helps turn the current supported AI conversation into an editable map.

## Local Data

TurnMap stores map data in the browser's extension storage:

- Conversation id and title
- Extracted question-answer turns
- Node positions, titles, summaries, tags, and statuses
- User-created and AI-suggested links
- Locally generated topic-analysis candidate links until the user accepts or clears them
- Layout and UI preferences
- AI provider settings and raw API key
- Custom UI language packs generated by AI or imported by the user
- Floating panel preferences
- Floating launcher position and visibility preferences
- Planned update notice preferences, such as ignored versions or reminder choices

This data stays in the local browser profile unless the user exports it or uses AI features.

## Local Topic Analysis

Analyze Topics runs in the extension UI using existing node titles, summaries, tags, node order, and saved links. It does not call a provider, does not create raw embedding vectors, and does not permanently change the graph until the user accepts candidate links.

## AI Provider Data

TurnMap does not call an AI provider automatically unless the user enables auto summarize. Local topic analysis is not an AI provider call.

When the user runs AI features, TurnMap sends the relevant node text, summaries, and map context to the configured provider:

- OpenAI
- DeepSeek
- OpenRouter
- Qwen / DashScope
- Kimi / Moonshot
- Doubao / Volcano Ark
- Zhipu / GLM
- Mistral
- Gemini-compatible Vertex-style endpoints
- A user-configured OpenAI-compatible endpoint

The provider receives the text needed for the requested feature, such as node summarization or link suggestion.

For AI UI translation, TurnMap sends only built-in UI labels and target language metadata. It does not send conversation text, node content, graph data, or exported files. If the model returns malformed JSON, TurnMap may send the model's UI-translation response back to the same configured provider once to repair the language pack format.

API keys are stored as the raw secret value in the local browser extension profile. TurnMap adds the `Bearer ` authorization prefix at request time and redacts key-like values from task logs and debug reports.

## Conversation Site Access

TurnMap reads the active supported AI conversation page to build the current conversation map. For ChatGPT, it may also read the ChatGPT conversation backend API using the user's existing ChatGPT session when that path is available.

The floating launcher runs only on supported AI conversation pages. It provides a quick way to open TurnMap or TurnMap settings and stores its visibility and position preferences locally.

## Update Notices

The planned Update Notice may check a GitHub Release page or a version manifest to determine whether a newer TurnMap version is available. This check should not include conversation text, API keys, or ChatGPT session data.

## Exports

Users can export maps as JSON, Markdown, OPML, Obsidian Canvas, Obsidian vault Markdown, SVG, or PNG. Users can also export custom UI language packs as JSON for local backup or community sharing. Exported files are controlled by the user.

## Permissions

TurnMap requests:

- `activeTab`, `tabs`, and `scripting` to communicate with the active supported AI conversation tab.
- `sidePanel` to provide the Edge side panel UI.
- `storage` to persist maps and settings locally.
- `webRequest` to support reliable ChatGPT conversation API extraction.
- Host access to supported AI conversation sites and built-in AI provider API hosts.
- Optional host access for custom AI endpoints, requested only when needed.
