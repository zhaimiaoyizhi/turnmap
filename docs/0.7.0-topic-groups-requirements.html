Topic Groups Module PRD

Owner scope: reversible one-level topic grouping with editable topic metadata and generated proxy links.

Requirements

  • Allow non-contiguous selected nodes to collapse into a semantic topic node.
  • Persist a long-lived topic group record with member IDs, original positions, restoration metadata, and topic node metadata.
  • Store only a minimal restoration snapshot: member positions, member hidden state, and related user-edge restoration data. Do not duplicate member node title, summary, source text, or sourceAnchors.
  • Topic node metadata includes title, summary, tags, color, importance, and display state.
  • While collapsed, represent external links touching grouped nodes with generated proxy links to/from the topic node.
  • Proxy links are generated view edges and must not become permanent user links.
  • Edge handling is based on group boundary, not user-vs-auto source. Edges with both endpoints inside the topic group are hidden while collapsed and restored on expansion. Edges crossing the group boundary become generated proxy links while collapsed and restore to their original endpoints on expansion. Edges entirely outside the group remain unchanged.
  • Expanding restores original nodes, original links, and layout.
  • After expansion, hide the topic node from the visible canvas and show restored member nodes plus original links. Keep topic metadata in the group record.
  • If the same member set is collapsed again later, reuse existing topic metadata instead of creating a duplicate group.
  • If member nodes are deleted after expansion, mark the old group record stale and do not auto-reuse it. A later collapse of the remaining nodes should create a new group rather than trying to repair the old member set.
  • Block nested topic collapse in 0.7.0.
  • Migrate old custom-topic notes only when the legacy id, topic tag, generated summary structure, and hiddenNodeIds provide high-confidence evidence.

Out Of Scope

  • Hidden group-management UI for one-click collapse back to an expanded topic.
  • Topic-in-topic nesting.
  • Replacing existing Analyze Topics candidate-link flow.

Acceptance

  • Tests cover non-contiguous groups, restore original nodes/edges/layout, editable metadata persistence, proxy link generation/removal, metadata reuse, nested-collapse blocking, and JSON save/restore.
  • Existing single-node fold behavior remains separate from topic grouping.

Risk: topic collapse already exists as hide selected nodes plus custom note. Because preview-era saved data is not the priority, 0.7.0 should migrate confidently identifiable old topic notes into proper topic group records when possible, and leave uncertain cases as ordinary notes.
